## 模块概览

`ai-chat-web` 是整个项目的 **前端单页应用（SPA）**，基于 Vue 3 + TypeScript + Vite 实现，主要职责是：

- 提供一个类似 ChatGPT 的聊天界面（会话列表、消息气泡、Markdown/代码高亮、截图导出等）。
- 通过 HTTP 接口与 `ai-chat-backend` 交互，发送用户输入、接收 AI 的流式回答。
- 管理本地状态（会话、设置、用户信息、提示词模板等），提供多语言与主题切换能力。

---

## 技术栈与关键依赖

- **框架与构建**
  - Vue 3 + `<script setup>`
  - TypeScript
  - Vite
- **状态管理与路由**
  - Pinia：集中管理聊天记录、设置、用户、认证等状态。
  - Vue Router：支持多页面/路由跳转（主要路由是聊天页和异常页）。
- **UI 与国际化**
  - Naive UI：组件库（按钮、输入框、对话框、自动完成等）。
  - Tailwind CSS：样式工具类。
  - vue-i18n：多语言支持（简体中文、繁体中文、英文）。
- **其他功能库**
  - axios：HTTP 请求封装。
  - markdown-it + highlight.js + katex：支持 Markdown 渲染、代码高亮与公式显示。
  - html2canvas：将聊天记录导出为图片。

## 目录结构与职责说明（核心部分）

- **入口与整体框架**
  - `src/main.ts`
    - 创建 Vue 应用，依次完成：
      - `setupAssets()`：引入全局样式等静态资源。
      - `setupScrollbarStyle()`：统一滚动条样式。
      - `setupStore(app)`：注册 Pinia。
      - `setupI18n(app)`：注册国际化。
      - `setupRouter(app)`：注册 Vue Router 并等待路由就绪。
    - 最终挂载到 `#app`。
  - `src/App.vue`
    - 使用 `NConfigProvider` 包裹全局：
      - 从 `useTheme` 获取当前主题与覆盖配置。
      - 从 `useLanguage` 获取当前语言。
    - 内部通过 `NaiveProvider` 包裹 `RouterView`，作为整个 SPA 的顶层布局。

- **视图层**
  - `src/views/chat/index.vue`
    - 主聊天页面，是用户主要交互的界面。
    - 核心功能：
      - 显示聊天消息列表（`Message` 组件）。
      - 输入框（支持多行、快捷键发送、Prompt 模板自动补全）。
      - 控制按钮：
        - 清空会话（`handleClear`）。
        - 导出聊天记录为图片（`handleExport`）。
        - 开关「使用上下文」模式（`toggleUsingContext`）。
        - 停止当前回答（`handleStop`，通过 `AbortController` 取消请求）。
      - 支持「重新生成回答」（`onRegenerate`）和删除单条消息（`handleDelete`）。
    - 管理逻辑：
      - 使用 `useChatStore` 获取当前会话数据。
      - 使用 `useChat` 封装的 `addChat`、`updateChat`、`updateChatSome`、`getChatByUuidAndIndex` 操作聊天列表。
      - 使用 `useScroll` 控制滚动条自动滚动到底部。
      - 使用 `useUsingContext` 管理是否携带上下文。
      - 使用 `usePromptStore` + `NAutoComplete` 提供 `/` 开头的 Prompt 模板联想。
    - 与后端交互：
      - 调用 `fetchChatAPIProcess`，在 `onDownloadProgress` 回调中解析 HTTP 流。
      - 本地逻辑会根据 `data.detail.choices[0].finish_reason === 'length'` 判断是否继续长回复。
  - `src/views/exception/404/index.vue`、`src/views/exception/500/index.vue`
    - 异常页，处理路由未命中或服务错误时的展示。

- **布局与组件**
  - `src/views/chat/layout/`
    - `Layout.vue`：聊天页面整体布局。
    - `sider/`：左侧会话列表、会话管理。
    - `SiderRight/`：右侧设置面板（如模型参数、系统提示词等）。
    - `Permission.vue`：与鉴权/权限相关的布局处理（如需要登录时的控制）。
  - `src/views/chat/components/`
    - `Message/`：单条消息组件（包括用户消息和 AI 消息，支持 Markdown 渲染、代码高亮等）。
    - `Header/`：移动端顶部标题栏，提供导出、上下文开关等入口。
  - `src/components/common/`
    - `NaiveProvider`：统一注册 Naive UI 提供的上下文（message、dialog 等）。
    - `HoverButton`：带 hover 效果的按钮包装。
    - `SvgIcon`：统一的图标组件。
    - `UserAvatar`、`PromptStore`、`Setting/*`：用户头像、Prompt 管理、设置面板等公共组件。
  - `src/components/custom/`
    - 如 `GithubSite.vue` 等自定义展示类组件。

- **状态管理**
  - `src/store/index.ts`、`src/store/modules/*`
    - `chat`：管理会话列表、消息列表等聊天相关状态。
    - `settings`：全局设置（如系统提示词、模型配置等）。
    - `user`：用户信息。
    - `auth`：认证状态与 Token。
    - `prompt`：Prompt 模板（快捷指令）列表。
    - `app`：应用级别状态（布局、主题相关辅助信息等）。

- **工具与封装**
  - `src/utils/request/axios.ts` + `src/utils/request/index.ts`
    - 对 axios 进行二次封装，统一处理：
      - 请求/响应拦截。
      - 统一的返回结构（`{ status, message, data }`）。
      - 401 未授权时的跳转逻辑（清除 Cookie 并跳转到登录中心）。
    - `http`、`get`、`post` 帮助方法，支持：
      - `onDownloadProgress`（用于处理流式响应）。
      - `beforeRequest` / `afterRequest` 钩子。
  - `src/api/index.ts`
    - 封装与后端的业务接口：
      - `fetchChatAPI`：一次性聊天接口（目前使用 `/chat`）。
      - `fetchChatConfig`：获取后端配置（`/config`）。
      - `fetchChatAPIProcess`：**流式聊天接口**（`/chat-process`），支持 `onDownloadProgress`。
      - `fetchSession`：获取会话/登录状态信息（`/session`）。
      - `fetchVerify`、`fetchCode`、`login`：用户认证与短信验证码相关接口。
  - 其他工具
    - `src/hooks/useTheme.ts`、`src/hooks/useLanguage.ts`：主题与语言控制。
    - `src/utils/storage/*`、`src/utils/cookie/*`：本地存储与 Cookie 操作。
    - `src/utils/crypto/index.ts`：加解密相关封装（例如对敏感数据进行处理）。

## 前后端交互方式分析

### 1. 与 `ai-chat-backend` 的基础约定

前端默认将所有请求发送到一个统一前缀（通常是 `/api`，由 Vite 或 Nginx 代理配置决定），`src/api/index.ts` 中配置的路径为：

- 聊天：
  - `POST /chat-process`：流式聊天接口（核心）。
  - `POST /chat`：一次性返回整段回答（可选）。
- 配置和会话：
  - `POST /config`：获取后端配置（模型信息等）。
  - `POST /session`：获取当前登录状态。
- 鉴权与登录：
  - `POST /verify`：校验 Token。
  - `POST /v1/sms/send/code`：发送短信验证码。
  - `POST /v1/user/login`：登录。

在 `ai-chat-backend` 中，`/api` 路由组会注册对应的 handler（如 `chat-process`、`config`、`session` 等），由 Gin 负责接收。

### 2. 流式聊天（重点）

- 前端调用：

  ```ts
  fetchChatAPIProcess<Chat.ConversationResponse>({
    prompt: message,
    options,
    signal: controller.signal,
    onDownloadProgress: ({ event }) => {
      const xhr = event.target
      const { responseText } = xhr
      const lastIndex = responseText.lastIndexOf('\n', responseText.length - 2)
      let chunk = responseText
      if (lastIndex !== -1)
        chunk = responseText.substring(lastIndex)
      const data = JSON.parse(chunk)
      // 使用 data 更新最新一条 AI 消息
    },
  })
  ```

- 协议约定：
  - 后端 `ai-chat-backend` 将 `Content-Type` 设为 `application/octet-stream`。
  - 每个增量结果写出一个 JSON 行，以 `\n` 分隔。
  - 前端通过 `onDownloadProgress` 获取当前累计的 `responseText`，截取最后一行，并 `JSON.parse` 还原为 `ChatMessage`。
  - 本质上是「HTTP chunked + 每行一个 JSON」，不是标准 SSE，但实现了类似效果。

### 3. 状态与配置交互

- 启动时，会通过：
  - `fetchChatConfig` 获取服务器侧当前模型、代理等配置。
  - `fetchSession` 获取会话状态（是否需要登录、Token 是否有效等）。
- 这些信息会写入 `settings` / `auth` / `user` 等 store 中，影响后续请求的 Header、路由权限等。

## 前端已实现的主要功能清单

- **聊天功能**
  - 多会话管理（左侧会话列表、新建、删除、清空）。
  - 聊天消息展示（包含时间、角色、错误状态、加载状态）。
  - 输入框支持：
    - 多行输入。
    - 桌面端 Enter 发送、Shift+Enter 换行；移动端 Ctrl+Enter 发送。
  - 流式回答：
    - 边生成边显示。
    - 支持中途「Stop Responding」终止回答。
  - 使用上下文：
    - 可通过按钮开关是否携带历史对话上下文。
  - 重试与错误处理：
    - 对单条回答支持「重新生成」。
    - 错误信息会拼接在最后一条消息中展示。

- **Prompt 模板**
  - 通过 `prompt` store 存储常用的 Prompt 模板。
  - 在输入框使用 `/` 开头时，自动联想并使用 `NAutoComplete` 提示。

- **聊天记录导出**
  - 使用 `html2canvas` 对 `#image-wrapper` 整个聊天区域截图。
  - 生成 PNG 图片并触发浏览器下载。

- **多语言与主题**
  - 多语言：支持中/英/繁，在设置中切换，文本通过 `t('xxx')` 获取。
  - 主题：支持浅色/深色主题切换，使用 `useTheme` 与 Naive UI 的主题系统。

- **认证与用户相关**
  - 支持通过短信验证码登录（`fetchCode` + `login`）。
  - 在收到 401 或特定状态码时，前端会：
    - 清理本地 Token / Cookie。
    - 跳转到统一用户中心地址（如 `https://user.0voice.com?sys=ai`）。

## 如何本地启动与调试

1. 进入前端目录：

   ```bash
   cd ai-chat-web
   ```

2. 安装依赖：

   ```bash
   pnpm install   # 或 npm install / yarn install
   ```

3. 启动开发服务器：

   ```bash
   pnpm dev       # 或 npm run dev / yarn dev
   ```

4. 打开浏览器访问终端提示的地址（如 `http://localhost:5173`），确保：
   - 代理配置正确，将 `/api` 请求转发到 `ai-chat-backend` 的服务地址。
   - `ai-chat-backend` 与 `ai-chat-service` 已按相应配置启动。

## 适合新手的阅读顺序建议

1. `src/main.ts` + `src/App.vue`：先弄清楚应用的启动流程与全局布局。
2. `src/views/chat/index.vue`：详细阅读聊天页面的逻辑，理解：
   - 如何读取/更新聊天 store。
   - 如何调用 `fetchChatAPIProcess` 实现流式聊天。
   - 如何处理导出、清空、重新生成等操作。
3. `src/api/index.ts` + `src/utils/request/index.ts`：搞清楚前端是如何调用后端接口的，以及流式响应的解析方式。
4. `src/store/modules/chat`、`src/store/modules/settings`、`src/store/modules/prompt`：了解会话、设置、Prompt 模板的数据结构。
5. 有余力时再看：
   - `src/views/chat/layout/*`：整体布局与侧边栏。
   - `src/views/chat/components/Message/*`：消息渲染与 Markdown 显示。

掌握以上内容后，你就能比较自信地在前端添加新按钮、新配置项，或调整与后端的交互方式（例如从自定义流协议迁移到 SSE）了。
